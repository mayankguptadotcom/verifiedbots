**Goal:** Grow the directory from 6 to 100+ accurately-sourced bots across all ten categories, add the Web Bot Auth key-directory CI check, and add the `json_array` feed format.

**Architecture:** Two small code tasks first (WBA validator + CLI wired into CI; `json_array` parser format), then seven category-batch curation tasks, each researching a fixed roster from operator documentation and the arcjet/well-known-bots reference, writing `bots/<id>.yaml` files that must pass the existing validator and live feed checks. **Tech Stack:** Existing TypeScript pipeline (vitest, ajv, ipaddr.js), GitHub Actions; curation is YAML data work driven by live web research.

Every `user_agents.instances` entry must be a real string from (a) the operator's official documentation, (b) the arcjet/well-known-bots `instances` list, or (c) well-attested public log corpora quoted in docs. - Categories enum: `search-engine`, `ai-crawler`, `ai-assistant`, `social-preview`, `monitoring`, `seo`, `feed-fetcher`, `archiver`, `security`, `webhook`.

`src/lib/wba.ts`:
```ts
import type { Bot } from './validateBot.js';

export type Fetcher = (url: string) => Promise<{ status: number; body: string }>;

export function validateKeyDirectoryBody(body: string): string[] {
  let doc: any;
  try {
    doc = JSON.parse(body);
  } catch {
    return ['key directory is not valid JSON'];
  }
  if (doc === null || typeof doc !== 'object' || !Array.isArray(doc.keys)) {
    return ['key directory has no "keys" array (expected an RFC 7517 JWK Set)'];
  }
  const issues: string[] = [];
  const ed = doc.keys.filter((k: any) => k && k.kty === 'OKP' && k.crv === 'Ed25519');
  if (ed.length === 0) issues.push('no Ed25519 (OKP) keys found — Web Bot Auth requires Ed25519');
  for (const k of ed) {
    if (typeof k.x !== 'string' || k.x.length === 0) issues.push('Ed25519 key missing base64url "x" coordinate');
  }
  return issues;
}

const defaultFetcher: Fetcher = async (url) => {
  const res = await fetch(url, { redirect: 'follow', signal: AbortSignal.timeout(30_000), headers: { 'user-agent': 'verifiedbots-pipeline/1.0' } });
  return { status: res.status, body: await res.text() };
};

export async function checkWba(bots: Bot[], fetcher: Fetcher = defaultFetcher): Promise<Array<{ botId: string; url: string; issues: string[] }>> {
  const rows: Array<{ botId: string; url: string; issues: string[] }> = [];
  for (const bot of bots) {
    for (const v of bot.verification) {
      if (v.type !== 'web_bot_auth') continue;
      const url = v.signature_agent_url;
      if (!url.startsWith('https://')) {
        rows.push({ botId: bot.id, url, issues: ['signature_agent_url must be https'] });
        continue;
      }
      try {
        const res = await fetcher(url);
        rows.push({ botId: bot.id, url, issues: res.status === 200 ? validateKeyDirectoryBody(res.body) : [`HTTP ${res.status}`] });
      } catch (e) {
        rows.push({ botId: bot.id, url, issues: [`fetch failed: ${e instanceof Error ? e.message : String(e)}`] });
      }
    }
  }
  return rows;
}
```

## Batch Curation Procedure (applies to Tasks 3–9; the FULL procedure is repeated inside each task's brief via this reference — implementers: follow it exactly)

Each batch task = research + write + validate + commit, for its fixed roster:

1. **Download references** (git-ignored scratch, e.g. `/tmp/`): `curl -sL https://raw.githubusercontent.com/arcjet/well-known-bots/main/well-known-bots.json -o /tmp/arcjet.json`. Read 2–3 existing `bots/*.yaml` as format exemplars.
2. **Per rostered bot, research in this order:** (a) operator's official bot-documentation page (WebFetch it — this is the primary source for UA strings, verification method, robots token); (b) arcjet entry (`jq '.[] | select(.id=="...")' /tmp/arcjet.json` or read the file) for UA instances/patterns; (c) the roster's "expected verification" hint — verify it live (curl the feed URL; confirm HTTP 200 and expected shape) before writing it into YAML.
3. **Write `bots/<id>.yaml`** following the existing file format exactly. Description: 1–2 factual sentences in your own words (never copy marketing copy). UA pattern: a minimal regex anchored on the bot token + `/\d` where versioned (mirror existing entries). Instances: real strings only, with source noted in your report. Verification: only what the operator documents; prefer `cidr_feed` (live-verified) > `rdns`/`asn`/`static_cidrs` > none (Tier 3 is acceptable and common for social/preview bots).
4. **Validate after every 3–4 files:** `npm run validate` (fix issues immediately). After the full roster: `npm run fetch-feeds` — EVERY feed must print OK; an ERROR line means a wrong URL/format — fix it or downgrade that bot's recipe. Then `npx vitest run` (full suite) and `npm run typecheck`.
5. **Report:** per bot — id, tier, verification types, and the source URL(s) for its UA strings and verification data. List any DROPPED bots with the reason. This provenance table is what the batch reviewer audits.
6. **Cleanup + commit:** `git checkout -- data/last-known-good` (discard fetch churn), then `git add bots/*.yaml` (explicit new files) and commit `feat(data): <category> batch — <n> bots`.

**Hard rules:** no invented data (drop instead); no feed URL committed without a live 200-OK check; no new categories; no edits to existing bot files unless the roster explicitly says so; never commit scratch downloads.

**Extras allowance:** a batch MAY add up to 3 bots beyond its roster in the same category IF each is fully sourced (operator docs page + real UA string + verification recipe or an explicit Tier-3 note). Every extra must be flagged `EXTRA` in the report's provenance table — reviewers audit extras first.
